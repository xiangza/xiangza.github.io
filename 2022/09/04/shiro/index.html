一、权限概述

1、什么是权限

权限管理：根据系统指定的安全规则，用户可以访问且只能访问自己被授权的资源。

权限管理一般分为：

  • 访问权限

    不同的身份角色，允许执行的操作不同，如淘宝卖家、买家

  • 数据权限

    一个角色身份，可以访问到的数据内容不同，如店铺主管可以看到手下客户人员所有服务的买家订单信息，而她的手下只能看到自己负责的订单信息

2、认证概念

1、什么是认证

身份认证，就是判断一个用户是否为合法用户的处理过程，最常用的简单身份认证方式就是，系统通过核对用户输入的用户名和口令，看其是否与系统存储的用户名和口令一致，来判断身份是否正确，如短信验证、密码验证

2、认证流程

3、关键对象

Subject：主体，访问系统的用户，主体可以是用户、程序等

Principal：身份信息是主体进行身份认证的标识，具有唯一性，如用户名、手机号、邮箱地址，一个主体可以有多个身份，但必须有一个主身份

credential：凭证信息：是只有主体自己才知道的安全信息，如密码、证书等

3、授权概念

1、什么是授权

授权、即访问控制，控制哪些身份的用户可以访问哪些资源，主体进行身份认证后，系统会为其分配对应的权限，当访问资源时，系统会校验对应用户是否有访问资源的权限。

用户对象user：当前操作的用户、程序

资源对象resource：当前被访问的对象

角色对象role：一组授权操作许可权的集合

权限对象permission：权限操作许可权

2、授权流程

3、关键对象

授权简单理解为who对what进行how操作

who：主体，可以是一个用户或程序

what：资源，如系统菜单、页面、按钮、方法、系统商品信息

  • 访问类型：商品菜单、订单菜单
  • 数据类型：我的商品、我的订单

how：权限许可

  • 我的商品 访问我的商品许可
  • 分销商菜单 访问分销商菜单许可

二、shiro概述

1、简介

什么是shiro：

一个开源框架，将软件系统的安全认证相关功能抽取出来，实现用户身份认证，权限校验、加密、会话管理等功能。组成的一个安全认证框架

特点：

  • 易于理解的java security API
  • 简单的身份认证，支持多种数据源（jdbc）
  • 对角色的的简单的鉴权（访问控制），也支持细粒度的鉴权
  • 支持一级缓存
  • 内置的基于pojo企业级会话管理，适用于web及非web
  • 异构客户端会话访问
  • 简单的加密API
  • 不与任何框架捆绑，可独立运行

2、核心组件

  • Subject

  • SecurityManager

  • Authenticator

  • Authorizer

  • Realm（数据库读取+认证功能+授权功能实现）

  • SessionManager

  • SessionDao

  • CacheManager

  • Cryptography

三、shiro入门

1、身份认证

1、基本流程

流程：

  1. Shiro把用户的数据封装成标识token，token一般封装着用户名，密码等信息
  2. 使用subject获取到封装着用户的token
  3. subject将token交给SecurityManager，在SecurityManager安全中心，SecurityManager把标识token委托给认证器Authenticator进行身份验证。认证器的作用一般是用来指定如何验证，它规定本次认证用到哪些Realm
  4. 认证器Authenticator将传入的标识token，与数据Realm对别，验证token是否合法

3、身份授权

shiro提供了base64和16进制的字符串编码、解码的api支持，方便一些编码解码操作

shiro内部的一些数据【存储/表示】都是用了base16和16进制字符串

1、基本流程

  1. 首先调用subject.isPermitted/hashRole接口，会委托给SecurityManager
  2. SecurityManager接着会委托给内部组件Authorizer
  3. authorizer在将请求委托给设置的realm去做，realm是真正的核心
  4. Realm将用户传入的请求参数封装成权限对象，在从数据库获取查询到的权限集合
  5. Realm将用户传入的权限对象，与从数据库中查出来的权限对象，进行比对。如果用户传入的权限对象在重数据库查出来的集合中，返回true。

进行权限操作的前提：身份认证成功
