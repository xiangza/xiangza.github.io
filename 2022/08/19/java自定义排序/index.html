Comparable和Comparator

java.lang包的Comparable接口 compareTo(Object obj) 和java.util包的Comparator接口 compare(Object obj1, Object obj2)都是集合中用来自定义比较的两个接口。

Comparator理解为外部比较器，而Comparable理解为内部比较器。

一、Comparable

java.lang包的Comparable是一个自定义对象实现自定义比较方法时需要实现的接口。如String、Integer等类都实现了Comparable接口。自定义要想加入集合后，能根据自己属性进行排序比较，可以通过实现此接口来完成。在用Collections类的sort方法排序时若不指定Comparator，那就以自然顺序排序。所谓自然顺序就是实现Comparable接口设定的排序方式。因此需要在实现了Comparable接口的类中重写comapreTo（T o）方法。

二、Comparator

java.util包的Comparator被称为外部比较器。当我们需要对某一个类A进行排序时，除了类A本身可以实现Comparable接口的方式外，还可以专门定义一个类B实现Comparator接口，来作为类A的比较器。这个比较器只需要实现Comparator接口即可。

注意

1、 类B即实现了Comparator接口的类，一定要实现compare（T o1， T o2）方法，该方法中就是我们自定义的对类A进行比较的规则 ；
2、 int compare(T o1, T o2) 是“比较o1和o2的大小”。返回“负数”，意味着“o1比o2小”；返回“零”，意味着“o1等于o2”；返回“正数”，意味着“o1大于o2”。

三、总结

Comparable可以直接在需要排序的类中实现compareTo（T o）。而Comparator需要另外定义一个比较器类。前者编写简单，但需要修改源代码；后者不需要修改源代码，而是另外实现一个比较器 当某个自定义的对象需要作比较的时候，把比较器和对象一起传递过去就可以比大小了， 并且在Comparator 里面用户可以自己实现复杂的可以通用的逻辑，使其可以匹配一些比较简单的对象，那样就可以节省很多重复劳动了。
